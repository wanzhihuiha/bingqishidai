# 当前项目 Agent Rules

> 这是一个给学员直接打开的 Godot + AI starter project。
> 每次开始工作前，先读 `docs/前置教学/README.md`、`docs/前置教学/01-环境配置.md` 和本文件。

## 1. 环境
- Windows | Godot 4.x
- 本机 Godot 4.6.2 命令行测试使用：`D:\DATA\ChromeDownload\Godot_v4.6.2-stable_win64.exe\Godot_v4.6.2-stable_win64_console.exe`。注意外层 `Godot_v4.6.2-stable_win64.exe` 是目录，不是真正的 exe。
- 使用中文回答 | UTF-8
- AI 工具： Codex

## 2. 工作方式
- 先读磁盘当前状态，再动手；不要假设文件已经存在。
- 优先让当前项目保持可运行，再继续加功能。
- 改完代码后，要明确告诉学员需要在 Godot 里怎么验证。
- 操作 `.tscn` 文件时要小心格式。如果 .tscn 反复出错，改用 GDScript 代码动态创建节点。
- 在 Codex 沙箱内直接运行 Godot 可能因无法写入 `user://logs` 触发 Godot 引擎崩溃；遇到这种情况，先使用提权运行验证，不要直接判断为项目脚本错误。

## 3. 项目边界
- 默认一直围绕当前这个项目文件夹工作，不引导学员切换到别的工作区。
- 教学文件统一放在 `docs/前置教学/`，后续正式项目文档会以一个独立的 `xx项目指导书/` 文件夹形式放到 `docs/` 下面。
- 未经用户明确允许，不修改、覆盖、重命名、重新生成或压缩 `docs/` 下的任何材料；需要理解资料时只读，确实需要改 `docs/` 时必须先列出改动范围并等待用户确认。
- 不再要求复制教学文档或重新建第二个项目。
- 不制造一次性脚手架、备份文件、占位垃圾文件。

## 4. 设计原则
- 回到问题本质，从源头解决，不在脏结构上叠补丁。
- 结构以必要为限；不为抽象、扩展或“以后可能有用”而新增层次。
- 使用 Godot 4.x 的新语法（@onready、@export、await 等），不要使用 Godot 3.x 的旧语法。
- Godot 4.6 GDScript 中，只有右侧类型非常明确时才使用 `:=`。从 Autoload、未显式类型变量、`Dictionary.get()`、JSON 数据、`DataManager` / `GameState` 查询结果取得值时，必须写显式类型，例如 `var quest_state: String = game_state.get_quest_state(quest_id)`、`var quest: Dictionary = data_manager.get_item("quests", quest_id)`、`var fuel_cost: int = int(entry_cost.get("fuel", 0))`，避免 F5 时脚本解析失败。

## 5. 项目专属规则

### 5.1 项目类型
- 本项目是原创冰原生存经营 + 单机轻 SLG 游戏。
- 设计重点围绕避难所经营、资源调度、幸存者岗位、英雄小队、冰原区域探索、事件推进和探险战斗结算展开。

### 5.2 原创边界
- 不复刻《无尽冬日》的角色、建筑名、UI、截图、音乐、联盟活动、商业化系统。
- 可以参考同类生存经营和轻 SLG 的通用设计方法，但具体命名、美术表现、界面布局、数值结构和玩法包装必须保持原创。
- 如需求中出现疑似复刻内容，先提醒风险，并改写为适合本项目的原创表达。

### 5.3 命名边界
- 后续文档、代码、场景、数据、资源、提交信息和对话汇报中，禁止使用旧四字项目名。
- 用户已确认新的项目显示名为“冰汽时代”。
- 旧四字项目名进入实际开发内容时，统一替换为“冰汽时代”；如不适合出现正式名称，则使用“本项目”“当前项目”等中性表达。
- 如历史资料或外部材料出现旧名，进入实际开发内容时统一改写为“本项目”“当前项目”“项目指导书”等中性表达，或使用用户另行确认的新项目名。

### 5.4 文件组织
- `data/` 放 JSON 数据。
- `scripts/managers/` 放全局管理器。
- `scripts/shelter/` 放避难所逻辑。
- `scripts/world/` 放冰原地图和区域逻辑。
- `scripts/battle/` 放探险战斗结算。
- `scripts/ui/` 放界面脚本。
- `scenes/` 按 `menu/`、`shelter/`、`world/`、`battle/`、`ui/` 分目录。
- `assets/` 按 `images/`、`videos/`、`audio/` 分目录。

### 5.5 数据原则
- 资源、建筑、岗位、英雄、区域、事件优先数据驱动。
- UI 不直接修改核心数据，只发出明确命令，由对应管理器或业务脚本处理。
- 战斗结算集中在结算器中，不散落在按钮回调或界面脚本里。
- JSON 数据读取后要显式转换关键字段类型，避免 Godot 4.6 GDScript 类型推断不稳定导致运行失败。

### 5.6 工作方式
- 修改功能前，先读 `docs/gdd.md`、`docs/plan.md`、`docs/data_schema.md`，再结合当前磁盘状态判断改动范围。
- 每次改动前，先说明方案和预计改动文件，等用户确认后再动手。
- 每完成一个可验证功能，就进行一次 git commit。
- 完成功能后，要说明在 Godot 里如何验证，并标明本次验证依赖的场景、按钮或操作路径。
